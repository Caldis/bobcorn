O que é EOT?

O EOT foi criado pela Microsoft em 1997 como o primeiro formato de fonte web da história. Foi submetido ao W3C para padronização, mas nunca foi aceito — principalmente devido ao seu algoritmo de compressão proprietário (MTX) e aos recursos de DRM integrados que outros fabricantes de navegadores rejeitaram.

O formato inclui vinculação de URL, um mecanismo que restringe as fontes para funcionar apenas em domínios específicos, e suporta subconjuntos de fontes para reduzir o tamanho do arquivo removendo glifos não utilizados. Esses eram recursos avançados para o final da década de 1990, mas a natureza proprietária do formato significou que ele só foi implementado no Internet Explorer.

Por quase uma década — do IE 4 até meados dos anos 2000 — o EOT era a única maneira de usar fontes personalizadas na web. Só quando o WOFF chegou em 2010 que existiu uma alternativa compatível com múltiplos navegadores. Hoje, o EOT é uma relíquia: útil apenas para projetos raros que ainda precisam suportar IE 8 ou anterior.

Como Funciona

Os arquivos EOT envolvem dados de fonte TrueType com a compressão MTX da Microsoft e um cabeçalho proprietário. O cabeçalho contém informações de vinculação de URL (uma lista de domínios onde a fonte tem permissão para carregar), sinalizadores de licenciamento e metadados da fonte, como nome da família e estilo.

Quando o Internet Explorer encontra uma fonte EOT em uma folha de estilos, lê o cabeçalho, verifica se a URL de origem da página atual corresponde a um dos domínios permitidos e somente então descomprime e renderiza a fonte. Se a verificação de URL falhar, a fonte é silenciosamente rejeitada — uma forma básica de proteção contra hotlink.

A sintaxe clássica "à prova de balas" do @font-face que inclui EOT é assim:

@font-face à prova de balas com EOT 
@font-face {
  font-family: 'MyIcons';
  src: url('icons.eot');                    /* IE9 compat */
  src: url('icons.eot?#iefix') format('embedded-opentype'),
       url('icons.woff2') format('woff2'),
       url('icons.woff') format('woff'),
       url('icons.ttf') format('truetype');
}

A query string ?#iefix é um artifício para contornar um bug do parser do IE 9. O IE 9 não conseguia tratar corretamente múltiplas entradas src em uma única declaração — ele tentava carregar a string inteira (incluindo as dicas de format()) como uma única URL, o que falharia. Adicionar ?#iefix à URL do EOT engana o IE 9 para tratá-la como uma URL válida e autônoma, encerrando o caminho antes que o parser engasgue com o restante da declaração. Os navegadores modernos ignoram completamente a query string.

Vantagens e Desvantagens

    Vantagens
  • Único formato de fonte suportado pelo IE 6-8
  • Tamanho de arquivo reduzido com compressão MTX
  • A vinculação de URL fornece proteção básica contra hotlink
    Desvantagens
  • Formato proprietário da Microsoft (nunca padronizado pelo W3C)
  • Funciona apenas no Internet Explorer
  • DRM complexo e vinculação de URL podem causar problemas de implantação
  • Nenhum navegador moderno oferece suporte
  • Tecnologia obsoleta sem desenvolvimento futuro

Quando Usar EOT

Somente quando você precisa obrigatoriamente suportar IE 8 ou anterior. Esses navegadores antigos não entendem WOFF, WOFF2, ou mesmo TTF bruto via @font-face — o EOT é a única opção para eles.

Para IE 9-11, o WOFF é a melhor escolha: é um padrão aberto, tem suporte mais amplo de ferramentas e não carrega o ônus do DRM e da vinculação de URL. Se seus dados analíticos mostram zero de tráfego IE 8 — o que é quase certo em 2024 e além — pule o EOT completamente.

Antes de incluir EOT em sua compilação, verifique seus dados analíticos reais de navegador. A participação global de IE 8 e versões anteriores é efetivamente de 0%. A menos que você esteja mirando em uma intranet corporativa específica ou sistema governamental preso a uma versão antiga do IE, o EOT adiciona complexidade à compilação sem nenhum benefício prático.

Aviso de legado
O Internet Explorer foi oficialmente descontinuado pela Microsoft em junho de 2022. O EOT existe exclusivamente para compatibilidade retroativa com sistemas extremamente legados (governos, intranets corporativas). Para todos os novos projetos, ignore o EOT.
No Bobcorn
O EOT é uma exportação opcional no Bobcorn. Ative-o apenas se seu projeto exigir explicitamente suporte ao IE 8 ou anterior. O Bobcorn utiliza a biblioteca ttf2eot para converter a partir de TTF, portanto não há etapa de conversão manual — basta ativar a opção no diálogo de exportação e o arquivo .eot será gerado junto com os demais formatos de fonte.
