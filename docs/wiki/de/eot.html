Was ist EOT?

EOT wurde 1997 von Microsoft als das erste Web-Schriftformat überhaupt entwickelt. Es wurde dem W3C zur Standardisierung eingereicht, aber nie akzeptiert — hauptsächlich wegen seines proprietären Komprimierungsalgorithmus (MTX) und der integrierten DRM-Funktionen, die andere Browseranbieter ablehnten.

Das Format enthält eine URL-Bindung, einen Mechanismus, der Schriften auf die Verwendung nur auf bestimmten Domains beschränkt, und unterstützt das Subsetting von Schriften, um die Dateigröße durch das Entfernen nicht verwendeter Glyphen zu reduzieren. Das waren zukunftsweisende Funktionen für die späten 1990er Jahre, aber die proprietäre Natur des Formats bedeutete, dass es nie in anderen Browsern als Internet Explorer implementiert wurde.

Für fast ein Jahrzehnt — von IE 4 bis Mitte der 2000er Jahre — war EOT der einzige Weg, benutzerdefinierte Schriften im Web zu verwenden. Erst als WOFF 2010 erschien, gab es eine browserübergreifende Alternative. Heute ist EOT ein Relikt: nur nützlich für die seltenen Projekte, die noch IE 8 oder früher unterstützen müssen.

Funktionsweise

EOT-Dateien umhüllen TrueType-Schriftdaten mit Microsofts MTX-Komprimierung und einem proprietären Header. Der Header enthält URL-Bindungsinformationen (eine Liste von Domains, auf denen die Schrift geladen werden darf), Lizenz-Flags und Schrift-Metadaten wie Familienname und Stil.

Wenn Internet Explorer eine EOT-Schrift in einem Stylesheet findet, liest er den Header, überprüft, ob die Ursprungs-URL der aktuellen Seite mit einer der erlaubten Domains übereinstimmt, und dekomprimiert und rendert die Schrift erst dann. Schlägt die URL-Prüfung fehl, wird die Schrift stillschweigend abgelehnt — eine grundlegende Form des Hotlink-Schutzes.

Die klassische „kugelsichere" @font-face-Syntax, die EOT enthält, sieht folgendermaßen aus:

Kugelsicheres @font-face mit EOT 
@font-face {
  font-family: 'MyIcons';
  src: url('icons.eot');                    /* IE9 compat */
  src: url('icons.eot?#iefix') format('embedded-opentype'),
       url('icons.woff2') format('woff2'),
       url('icons.woff') format('woff'),
       url('icons.ttf') format('truetype');
}

Der Query-String ?#iefix ist ein Hack, um einen Parser-Bug in IE 9 zu umgehen. IE 9 konnte mehrere src-Einträge in einer einzigen Deklaration nicht korrekt verarbeiten — es versuchte, den gesamten String (einschließlich der format()-Hinweise) als einzelne URL zu laden, was fehlschlug. Das Hinzufügen von ?#iefix zur EOT-URL bringt IE 9 dazu, sie als gültige, eigenständige URL zu behandeln, indem der Pfad beendet wird, bevor der Parser am Rest der Deklaration scheitert. Moderne Browser ignorieren den Query-String vollständig.

Vor- und Nachteile

    Vorteile
  • Einziges Schriftformat, das von IE 6-8 unterstützt wird
  • Kleine Dateigröße durch MTX-Komprimierung
  • URL-Bindung bietet grundlegenden Hotlink-Schutz
    Nachteile
  • Proprietäres Microsoft-Format (nie vom W3C standardisiert)
  • Funktioniert nur in Internet Explorer
  • Komplexe DRM/URL-Bindung kann Deployment-Probleme verursachen
  • Kein moderner Browser unterstützt es
  • Veraltete Technologie ohne künftige Weiterentwicklung

Wann EOT verwenden

Nur wenn Sie IE 8 oder früher unterstützen müssen. Diese veralteten Browser verstehen weder WOFF, WOFF2 noch rohes TTF über @font-face — EOT ist ihre einzige Option.

Für IE 9-11 ist WOFF die bessere Wahl: Es ist ein offener Standard, hat breitere Tool-Unterstützung und bringt nicht das Gepäck von DRM und URL-Bindung mit sich. Wenn Ihre Analysen keinen IE 8-Datenverkehr zeigen — was im Jahr 2024 und darüber hinaus mit hoher Wahrscheinlichkeit der Fall ist — lassen Sie EOT vollständig weg.

Prüfen Sie vor der Aufnahme von EOT in Ihren Build Ihre tatsächlichen Browser-Analysen. Der weltweite Nutzungsanteil von IE 8 und darunter ist effektiv 0 %. Sofern Sie kein spezifisches Unternehmens-Intranet oder Regierungssystem ansprechen, das auf einer alten IE-Version feststeckt, fügt EOT Build-Komplexität ohne praktischen Nutzen hinzu.

Legacy-Hinweis
Internet Explorer wurde von Microsoft offiziell im Juni 2022 eingestellt. EOT existiert ausschließlich für die Abwärtskompatibilität mit extrem veralteten Systemen (Regierung, Unternehmens-Intranets). Für alle neuen Projekte: EOT weglassen.
In Bobcorn
EOT ist ein optionaler Export in Bobcorn. Aktivieren Sie es nur, wenn Ihr Projekt ausdrücklich IE 8 oder früher erfordert. Bobcorn verwendet die ttf2eot-Bibliothek zur Konvertierung aus TTF, sodass kein manueller Konvertierungsschritt erforderlich ist — schalten Sie einfach die Option im Export-Dialog um und die .eot-Datei wird neben Ihren anderen Schriftformaten generiert.
