Apa Itu EOT?

EOT dibuat oleh Microsoft pada tahun 1997 sebagai format web font pertama yang pernah ada. Ini diajukan ke W3C untuk standardisasi tetapi tidak pernah diterima — sebagian besar karena algoritma kompresi proprietary-nya (MTX) dan fitur DRM bawaan yang ditolak oleh vendor browser lain.

Format ini mencakup URL binding, sebuah mekanisme yang membatasi font agar hanya berfungsi pada domain tertentu, dan mendukung subsetting font untuk mengurangi ukuran file dengan menghapus glyph yang tidak digunakan. Ini adalah fitur-fitur yang maju untuk akhir 1990-an, tetapi sifat proprietary dari format tersebut berarti ia hanya pernah diimplementasikan di Internet Explorer.

Hampir satu dekade — dari IE 4 hingga pertengahan 2000-an — EOT adalah satu-satunya cara untuk menggunakan font kustom di web. Baru pada saat WOFF tiba pada tahun 2010 ada alternatif lintas browser. Hari ini, EOT adalah relik: berguna hanya untuk proyek langka yang masih harus mendukung IE 8 atau sebelumnya.

Cara Kerjanya

File EOT membungkus data font TrueType dengan kompresi MTX Microsoft dan header proprietary. Header berisi informasi URL binding (daftar domain tempat font diizinkan untuk dimuat), flag lisensi, dan metadata font seperti nama family dan style.

Ketika Internet Explorer menemukan EOT font dalam stylesheet, ia membaca header, memverifikasi bahwa URL asal halaman saat ini cocok dengan salah satu domain yang diizinkan, dan hanya setelah itu mendekompresi dan merender font. Jika pemeriksaan URL gagal, font ditolak secara diam-diam — sebuah bentuk perlindungan hotlink dasar.

Sintaks "bulletproof" @font-face klasik yang menyertakan EOT terlihat seperti ini:

Bulletproof @font-face dengan EOT 
@font-face {
  font-family: 'MyIcons';
  src: url('icons.eot');                    /* IE9 compat */
  src: url('icons.eot?#iefix') format('embedded-opentype'),
       url('icons.woff2') format('woff2'),
       url('icons.woff') format('woff'),
       url('icons.ttf') format('truetype');
}

Query string ?#iefix adalah hack untuk mengatasi bug parser IE 9. IE 9 tidak dapat menangani beberapa entri src dalam satu deklarasi dengan benar — ia akan mencoba memuat seluruh string (termasuk petunjuk format()) sebagai satu URL, yang akan gagal. Menambahkan ?#iefix ke URL EOT menipu IE 9 untuk memperlakukannya sebagai URL yang valid dan mandiri. Browser modern mengabaikan query string sepenuhnya.

Kelebihan & Kekurangan

    Kelebihan
  • Satu-satunya format font yang didukung IE 6-8
  • Ukuran file kecil dengan kompresi MTX
  • URL binding memberikan perlindungan hotlink dasar
    Kekurangan
  • Format proprietary Microsoft (tidak pernah distandarisasi oleh W3C)
  • Hanya berfungsi di Internet Explorer
  • DRM/URL binding yang kompleks dapat menyebabkan masalah deployment
  • Tidak ada browser modern yang mendukungnya
  • Teknologi yang sudah usang tanpa pengembangan masa depan

Kapan Menggunakan EOT

Hanya ketika Anda harus mendukung IE 8 atau sebelumnya. Browser kuno ini tidak memahami WOFF, WOFF2, atau bahkan TTF mentah melalui @font-face — EOT adalah satu-satunya pilihan mereka.

Untuk IE 9-11, WOFF adalah pilihan yang lebih baik: ini adalah standar terbuka, memiliki dukungan alat yang lebih luas, dan tidak membawa beban DRM dan URL binding. Jika analitik Anda menunjukkan nol traffic IE 8 — yang sangat mungkin terjadi di tahun 2024 dan seterusnya — lewati EOT sepenuhnya.

Sebelum menyertakan EOT dalam build Anda, periksa analitik browser Anda yang sebenarnya. Pangsa penggunaan global IE 8 dan di bawahnya secara efektif adalah 0%. Kecuali Anda menargetkan intranet perusahaan tertentu atau sistem pemerintahan yang terkunci pada versi IE kuno, EOT menambah kompleksitas build tanpa manfaat praktis.

Pemberitahuan warisan
Internet Explorer secara resmi dihentikan oleh Microsoft pada Juni 2022. EOT ada semata-mata untuk kompatibilitas mundur dengan sistem yang sangat lama (pemerintahan, intranet perusahaan). Untuk semua proyek baru, lewati EOT.
Di Bobcorn
EOT adalah ekspor opsional di Bobcorn. Aktifkan hanya jika proyek Anda secara eksplisit memerlukan IE 8 atau sebelumnya. Bobcorn menggunakan library ttf2eot untuk mengkonversi dari TTF, sehingga tidak ada langkah konversi manual — cukup aktifkan opsi dalam dialog ekspor dan file .eot akan dihasilkan bersama format font Anda yang lain.
