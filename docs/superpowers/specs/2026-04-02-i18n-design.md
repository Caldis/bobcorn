# i18n 多语言支持设计

> 为 Bobcorn 引入完整的应用级多语言支持，覆盖 renderer 和主进程。

## 决策摘要

| 决策 | 选择 |
|------|------|
| 框架 | i18next + react-i18next |
| 首批语言 | 中文 (zh-CN) + 英文 (en)，社区可扩展 |
| 切换方式 | 跟随系统语言 + 手动覆盖（设置中） |
| 覆盖范围 | 全量（renderer UI + 主进程菜单 + 对话框） |
| Key 风格 | 扁平点分命名空间语义 key（`menu.file.save`） |
| 文件格式 | 单 JSON 文件 per 语言 |
| 回退策略 | `fallbackLng: 'zh-CN'`，缺失 key 显示中文 |

## 1. 架构

### 双进程独立实例

主进程和渲染进程各自初始化 i18next 实例，加载同一份翻译 JSON。

```
locales/zh-CN.json  locales/en.json
       │                   │
  ┌────▼────┐         ┌────▼──────────┐
  │  Main   │         │   Renderer    │
  │ i18next │◄──IPC──►│ i18next +     │
  │ 独立实例 │         │ react-i18next │
  └────┬────┘         └────┬──────────┘
       │                   │
  原生菜单/对话框      React 组件 useTranslation()
```

- 翻译文件通过 ES import 打入 bundle，不需要运行时文件读取
- 语言偏好存 `localStorage`，主进程通过 IPC 同步
- 语言切换时 renderer 自动 re-render，主进程重建菜单

## 2. 文件结构

### 新增文件

```
src/
├── locales/
│   ├── zh-CN.json       # 中文翻译（~150 keys）
│   ├── en.json           # 英文翻译
│   └── index.js          # 导出 resources + supportedLanguages
├── renderer/
│   └── i18n.js           # Renderer i18next 初始化
└── main/
    └── i18n.js           # Main process i18next 初始化
```

### 翻译文件格式

扁平 key + 点分命名空间，i18next `keySeparator: '.'` 默认支持：

```json
{
  "menu.file": "文件",
  "menu.file.new": "新建项目",
  "menu.file.save": "保存",
  "export.title": "导出图标字体",
  "export.progress.preparing": "准备导出 {{count}} 个图标...",
  "export.success": "✓ 导出完成！共 {{count}} 个文件",
  "batch.selected": "已选中 {{count}} 个图标",
  "batch.moved": "已移动 {{count}} 个图标",
  "common.cancel": "取消",
  "common.confirm": "确认",
  "common.save": "保存",
  "settings.language": "语言",
  "settings.followSystem": "跟随系统"
}
```

### locales/index.js

```js
import zhCN from './zh-CN.json'
import en from './en.json'

export const resources = {
  'zh-CN': { translation: zhCN },
  'en': { translation: en }
}

export const supportedLanguages = [
  { code: 'zh-CN', label: '简体中文' },
  { code: 'en', label: 'English' }
]
```

社区贡献者添加语言：新增 JSON + 在此注册。

## 3. Renderer 集成

### 初始化 (src/renderer/i18n.js)

```js
import i18n from 'i18next'
import { initReactI18next } from 'react-i18next'
import { resources } from '../locales'

const savedLng = localStorage.getItem('language')

i18n.use(initReactI18next).init({
  resources,
  lng: savedLng || undefined,
  fallbackLng: 'zh-CN',
  interpolation: { escapeValue: false }
})

export default i18n
```

在 `entry.js` 顶部 `import './i18n'` 确保在 React 挂载前初始化。

### 组件使用

```jsx
import { useTranslation } from 'react-i18next'

function ExportDialog() {
  const { t } = useTranslation()
  return (
    <Dialog title={t('export.title')}>
      <p>{t('export.progress.preparing', { count: icons.length })}</p>
      <Button>{t('common.cancel')}</Button>
    </Dialog>
  )
}
```

无需 Provider 包裹 — react-i18next v12+ 默认使用全局实例。

## 4. 主进程集成

### 初始化 (src/main/i18n.js)

```js
import i18n from 'i18next'
import { resources } from '../locales'
import { app } from 'electron'

const mainI18n = i18n.createInstance()
mainI18n.init({
  resources,
  lng: app.getLocale(),
  fallbackLng: 'zh-CN',
  interpolation: { escapeValue: false }
})

export default mainI18n
```

### 菜单重建

`menu.js` 改造为导出 `buildMenu()` 函数，内部用 `i18n.t()` 构建标签：

```js
import i18n from './i18n'

export function buildMenu() {
  const t = i18n.t.bind(i18n)
  const template = [
    {
      label: t('menu.file'),
      submenu: [
        { label: t('menu.file.new'), accelerator: 'CmdOrCtrl+N', ... },
        { label: t('menu.file.save'), accelerator: 'CmdOrCtrl+S', ... },
      ]
    },
  ]
  Menu.setApplicationMenu(Menu.buildFromTemplate(template))
}
```

## 5. 语言切换同步

### IPC 协议

```
Renderer → Main:  'language-changed' (lng: string)
Main → Renderer:  'request-language' ()
```

### Preload 新增

```js
languageChanged: (lng) => ipcRenderer.send('language-changed', lng),
onLanguageRequest: (callback) => {
  const handler = () => callback()
  ipcRenderer.on('request-language', handler)
  return () => ipcRenderer.removeListener('request-language', handler)
}
```

### 启动时序

1. 主进程启动 → `app.getLocale()` 初始化（临时值）
2. Renderer 加载 → 读 `localStorage.language` 初始化
3. `did-finish-load` → 主进程发 `request-language`
4. Renderer 回复 → 主进程 `changeLanguage()` + `buildMenu()`

### 切换流程

1. 用户在设置中选择语言
2. `i18n.changeLanguage(lng)` — React 自动 re-render
3. `localStorage.setItem('language', lng)`
4. `ipcRenderer.send('language-changed', lng)`
5. 主进程 `i18n.changeLanguage(lng)` + `buildMenu()`

## 6. 设置 UI

在 PrefixDialog（或重命名为 SettingsDialog）中新增语言选择区域：

```
┌─ 设置 ─────────────────────────┐
│                                │
│  图标字体前缀                    │
│  [icon-       ]                │
│                                │
│  ─────────────────────────     │
│                                │
│  语言 / Language                │
│  [简体中文          ▾]          │
│    ☐ 跟随系统                   │
│                                │
│          [取消]  [保存]          │
└────────────────────────────────┘
```

- 下拉列表从 `supportedLanguages` 动态渲染
- 「跟随系统」checkbox 勾选后禁用下拉，清除 `localStorage.language`
- 切换即时生效，无需重启

## 7. 迁移策略

### 迁移顺序（按风险从低到高）

```
Phase 1: 基础设施
  安装依赖、创建翻译文件、初始化 i18n（行为不变）

Phase 2: 通用字符串 (common.*)
  取消、确认、保存、删除等高频复用字符串

Phase 3: 按模块逐步迁移
  3a. 欢迎页 (splash.*)           ~7 strings
  3b. 导航栏 (nav.*)              ~6 strings
  3c. 分组管理 (group.*)          ~10 strings
  3d. 图标编辑器 (editor.*)       ~25 strings
  3e. 批量操作 (batch.*)          ~18 strings
  3f. 导出对话框 (export.*)       ~50 strings
  3g. 文件操作 (file.*)           ~12 strings
  3h. 工具栏 (toolbar.*)          ~20 strings
  3i. 设置 (settings.*)           ~5 strings (新增)

Phase 4: 主进程菜单迁移 (menu.*)
  menu.js 改造为 buildMenu() + i18n.t()

Phase 5: 文档与工具
  README 贡献指南 + 翻译完整性检查脚本
```

### 迁移模式

```jsx
// Before
<Button>取消</Button>
message.success(`已移动 ${count} 个图标`)

// After
<Button>{t('common.cancel')}</Button>
message.success(t('batch.moved', { count }))
```

每个模块迁移后运行 `npx vitest run` 确保不破坏测试。
中文 JSON 值 = 当前硬编码字符串，迁移后用户体验零变化。

## 9. 社区贡献流程

添加新语言 3 步：

1. 复制 `src/locales/zh-CN.json` → `src/locales/xx.json`
2. 翻译所有值（key 不动）
3. 在 `src/locales/index.js` 注册语言

此流程需同步写入 README 的贡献指南章节。

## 10. 不在本次范围

以下不实现，但架构不阻塞：

- 复数规则 — i18next 内置支持，`key_one`/`key_other` 后缀
- RTL 布局 — CSS logical properties
- 翻译管理平台 — Crowdin / Weblate
- 日期/数字本地化格式 — 当前无此需求
